FAILURE means:
  + Exit status: >0
  + Failing target will not be modified (but a tempfile will be created).
  + Failing target's metadata will not be modified.
  + Nota Bene: Successful dependencies will be built even on error.

SUCCESS means:
  + Exit status: 0

------------------------------------------------------------------------------

Attempt to build target when there is no source dir.

* FAILURE

------------------------------------------------------------------------------

Attempt to build target when there is no script dir.

* FAILURE

------------------------------------------------------------------------------

Attempt to build target when there is no build script for it.

* FAILURE

No files should be created. Not even 'TARGET--fixing'

------------------------------------------------------------------------------

Attempt to build when no previous target exist and build script returns
non-zero exit status.

* FAILURE

------------------------------------------------------------------------------

Attempt to build when no previous target exist and build script returns exit
status zero.

* SUCCESS

------------------------------------------------------------------------------

Build target when previous target exist but target + buildscript are unmodified
since last build

* SUCCESS: old target kept (neither metadata nor target modified)

------------------------------------------------------------------------------

Build target when previous target exist and target is unmodified since last
build, but buildscript now output something new

* SUCCESS: target + metadata modified

------------------------------------------------------------------------------

Build target when previous target exist and is modified

* fail (neither metadata nor target modified)

------------------------------------------------------------------------------

Build target when previous target exist and is modified, but new buildscript
outputs target identical to modified target

* SUCCESS: old target kept (metadata updated)

------------------------------------------------------------------------------

Build target A with dependency B

------------------------------------------------------------------------------

Build target when previous target exist but have no metadata stored, and the
newly built target differs from the old target

* FAIL

------------------------------------------------------------------------------

Build target when previous target exist but have no metadata stored, and the
newly built target is the same as the old target

* SUCCESS
